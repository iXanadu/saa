# Project Specifications

Place your project requirements and specifications in this folder.

## Suggested Files

- **prd.md** - Product Requirements Document (main project spec)
- **features.md** - Detailed feature specifications
- **technical.md** - Technical requirements and constraints
- **api.md** - API specifications (if applicable)

## PRD Template

Your `prd.md` should include:

1. **Project Overview** - What is this application?
2. **Target Users** - Who will use it?
3. **Core Features** - What must it do? (MVP)
4. **Technical Requirements** - Stack, integrations, constraints
5. **Data Models** - Key entities and relationships
6. **User Flows** - Main user journeys
7. **Future Features** - Post-MVP enhancements

## Usage

When starting a new session, Claude will read these specs to understand project requirements and make informed implementation decisions.
